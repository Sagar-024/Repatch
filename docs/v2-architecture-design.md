## Problem Statement
The current agent (Repatch) operates like a "script" that brute-forces file searching and tool execution. This leads to two major issues:
1. **Path Mapping Hallucination:** The agent often guesses file paths based on issue reports (e.g., `hermes/agent/`) that don't match the actual repository structure (e.g., `agent/`).
2. **Opacity (The Black Box):** The user sees tool calls but doesn't understand the "intent" or "hypothesis" behind them, reducing trust and making the agent feel less like a senior engineer.

## Demand Evidence
Founders and senior engineers value "correctness" and "reasoning" over raw speed. The "Inviolable Loop" is already a strong signal, but it needs a better "Mental Map" and "Voice" to be world-class.

## Status Quo
- **Path Mapping:** `resolvePath` in `machine.ts` is basic and relies on the agent guessing correctly.
- **Transparency:** Simple `console.log` of tool names. No internal monologue or intent streaming.

## Target User & Narrowest Wedge
**Target User:** Startup Founders and Engineering Managers.
**Narrowest Wedge:** A "10-star" demonstration video showing the agent intelligently navigating a messy repo and explaining its reasoning in real-time.

## Premises
1. **Context is King:** The agent cannot navigate effectively without a "Map of Truth."
2. **Transparency = Trust:** Users will wait for a correct fix if they can see the logic behind the process.
3. **Engineering Maturity:** Standardized "Intent Streaming" separates toy agents from production tools.

## Cross-Model Perspective (Synthesis)
Both the user and the assistant agree that "Semantic Path Indexing" and "Thought-Streaming" are the industry-standard solutions used by top-tier products like Cursor and GitHub Copilot Workspace.

## Recommended Approach: The Senior Operator
We will implement two core architectural upgrades:

### 1. Semantic Path Indexing (The Map of Truth)
- **Implementation:** The Orchestrator will run a recursive file listing (ignoring `node_modules`, `.git`, etc.) at startup.
- **Injection:** This compressed "File Tree Index" will be injected into the System Prompt.
- **Constraint:** The agent will be instructed to ALWAYS verify paths against the Index before attempting a `read_file`.

### 2. Thought-Streaming (The Internal Monologue)
- **Implementation:** Update the `AgentState` and `Step` interfaces to include a `thought` or `monologue` field.
- **Execution:** The LLM will be required to output its "Thought" (Hypothesis + Action) in every turn.
- **UI/UX:** The Orchestrator will stream this thought to the console with distinct styling (e.g., `💭 Thinking: ...`) BEFORE executing tools.

## Success Criteria
- Agent correctly maps `hermes/agent/file.ts` to `agent/file.ts` without multiple `ls` calls.
- User can follow the agent's logic through a clear, human-readable monologue in the terminal.

## The Assignment
Implement the "Map of Truth" generator in `src/orchestrator/utils.ts` and integrate it into the `Understand` step.
