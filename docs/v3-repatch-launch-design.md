## Problem Statement
Software maintainers and developers are drowning in bug reports. The "Status Quo" is a manual, soul-crushing grind where reproducing a bug often takes longer than fixing it. Currently, Repatch is a "vitamin"—an impressive engineering feat that hasn't yet found its "painkiller" application.

## Demand Evidence
*   **The Pain:** Developers waste 30–60% of their time just trying to get a bug to happen on their machine.
*   **Market Validation:** People pay for commercial AI coding tools, but those tools often hallucinate or fail on complex environment setups. 
*   **The Pull:** A tool that reliably delivers a **verifiable reproduction test** (Reproduction-as-a-Service) is an acute painkiller for maintainers with large backlogs.

## Status Quo
Manual reproduction: Reading issues -> Setting up environments -> Writing tests -> Debugging. This costs hours per bug and leads to maintainer burnout.

## Target User & Narrowest Wedge
*   **Target User:** Solo maintainers of widely-used open-source libraries. They are overwhelmed, guilt-ridden, and need to prioritize their "brain-cycles."
*   **Narrowest Wedge:** **Automated Bug Reproduction.** A tool that does *only* one thing: given a repo and an issue, it returns a failing test case in a sandbox. It builds trust by not touching source code.

## Constraints
*   **Trust is the Barrier:** Maintainers won't let AI edit code until they trust it.
*   **Environment Complexity:** Projects have wildly different runtimes (Node, Python, Go).
*   **Security:** AI execution must be isolated.

## Premises
1.  **Reproduction is the Wedge:** Saving reproduction time is more sellable than "fixing everything."
2.  **Trust is Earned:** Starting with "read-only" reproduction is the path to "write-access" fixing.
3.  **Infrastructure is the Moat:** Docker + Nixpacks provides the reliability that LLM wrappers lack.

## Recommended Approach: The Power CLI
A standalone binary/npm package that focuses exclusively on the `reproduce` command.
*   **Input:** `repatch reproduce <url> <issue#>`
*   **Output:** A `repro.test.ts` (or equivalent) and a terminal log of the failure.
*   **Why:** It respects the maintainer's control, runs locally, and delivers immediate value without risk.
